# Plan: target-specific file extensions

## Design

Every Beagle source file uses an extension that declares its target.
The `#lang` header must match. Mismatch is a hard error.

```
.bgl    → #lang beagle       (portable core)
.bclj   → #lang beagle/clj
.bcljs  → #lang beagle/cljs
.bjs    → #lang beagle/js
.bnix   → #lang beagle/nix
.bsql   → #lang beagle/sql
.bpy    → #lang beagle/py
```

No backward compatibility. `.bgl` means portable only. Existing
`.bgl` files with target-specific `#lang` lines get renamed.
`.rkt` still accepted (Racket compatibility) but not validated.

## Invariant

```
file extension = routing hint
#lang line     = authoritative compiler mode
mismatch       = hard error
```

## What to change

### 1. Extension-to-target map (new)

Define `EXTENSION-TARGET-MAP` somewhere accessible (e.g. a shared
module or inline in reader/parse):

```racket
(define EXTENSION-TARGET-MAP
  '((".bgl"   . #f)      ; portable — any target ok, or core
    (".bclj"  . clj)
    (".bcljs" . cljs)
    (".bjs"   . js)
    (".bnix"  . nix)
    (".bsql"  . sql)
    (".bpy"   . py)
    (".rkt"   . #f)))    ; legacy — no validation
```

And the reverse for output:

```racket
(define TARGET-OUTPUT-EXT
  '((clj  . ".clj")
    (cljs . ".cljs")
    (js   . ".js")
    (nix  . ".nix")
    (sql  . ".sql")
    (py   . ".py")))
```

### 2. Validation in parse/compile pipeline

In `main.rkt` or the parse entry point, after reading the `#lang`
line and determining the target, validate against the file extension:

```racket
(define (validate-extension! source-path target)
  (define ext (path-get-extension source-path))
  (define expected (assoc ext EXTENSION-TARGET-MAP))
  (when (and expected (cdr expected)  ; not .rkt or .bgl
              (not (eq? target (cdr expected))))
    (error 'beagle
           "extension/header mismatch: ~a expects #lang beagle/~a, found #lang beagle/~a"
           ext (cdr expected) target)))
```

For `.bgl`: if the target is anything other than the portable core
(once that exists), error. For now, `.bgl` with `define-target`
could be an error or a warning — depends on whether we want to force
rename immediately.

### 3. Rename existing files

373 total `.bgl` files. By target:
- 28 are `beagle/nix` → rename to `.bnix`
- 5 are `beagle/js` → rename to `.bjs`
- 10 are `beagle/cljs` → rename to `.bcljs`
- ~330 are `beagle/clj` (default) → rename to `.bclj`
- Remainder with no explicit target → stay `.bgl` or become `.bclj`

Script:
```bash
for f in $(find . -name '*.bgl' -not -path '*/.git/*'); do
  if grep -q 'beagle/nix\|define-target nix' "$f"; then
    mv "$f" "${f%.bgl}.bnix"
  elif grep -q 'beagle/js\|define-target js' "$f"; then
    mv "$f" "${f%.bgl}.bjs"
  elif grep -q 'beagle/cljs\|define-target cljs' "$f"; then
    mv "$f" "${f%.bgl}.bcljs"
  elif grep -q 'beagle/sql\|define-target sql' "$f"; then
    mv "$f" "${f%.bgl}.bsql"
  else
    mv "$f" "${f%.bgl}.bclj"
  fi
done
```

### 5. Define a helper

To avoid repeating the extension list everywhere, define one
canonical helper:

```racket
(define BEAGLE-EXTENSIONS '(".bgl" ".bclj" ".bcljs" ".bjs" ".bnix" ".bsql" ".bpy" ".rkt"))

(define (beagle-source-file? path)
  (define ext (path-get-extension path))
  (and ext (member (bytes->string/utf-8 ext) BEAGLE-EXTENSIONS) #t))
```

All file-matching code uses this instead of hardcoding extensions.
